<task type="auto">
  <name>Task 1: Wire sublane, social force, spatial index, and swarming color into sim.rs</name>
  <files>crates/velos-gpu/src/sim.rs</files>
  <action>
Modify `crates/velos-gpu/src/sim.rs` to integrate all Phase 3 behaviors. This is a single large task because all changes are in one file and interdependent.

**1. Add imports:**
```rust
use velos_core::components::LateralOffset;
use velos_vehicle::sublane::{self, SublaneParams, NeighborInfo};
use velos_vehicle::social_force::{self, SocialForceParams, PedestrianNeighbor};
use velos_net::SpatialIndex;
```

**2. Add fields to SimWorld:**
- `sublane_params: SublaneParams` (Default::default())
- `social_force_params: SocialForceParams` (Default::default())
Initialize both in `new()` and `reset()`.

**3. Modify `spawn_single_agent()`:**
When vtype is Motorbike, also spawn a `LateralOffset { lateral_offset: road_width / 2.0, desired_lateral: road_width / 2.0 }` component. Road width = lane_count * 3.5. Look up the edge's lane_count from road_graph. Cars and pedestrians do NOT get LateralOffset. This requires two spawn paths: one with LateralOffset tuple for motorbikes, one without for others.

**4. Create `AgentSnapshot` struct and `collect_all_agents()` helper:**
Build once per frame: query ALL entities with (Position, Kinematics, VehicleType). Return struct with parallel vecs: ids (entity id as u32), positions ([f64;2]), vehicle_types, speeds, lateral_offsets (Option<f64>). This is the frame's snapshot for spatial queries.

**5. Modify `tick()` call order:**
```rust
self.spawn_agents(dt);
self.step_signals(dt);
let snapshot = self.collect_all_agents();
let spatial = SpatialIndex::from_positions(&snapshot.ids, &snapshot.positions);
self.step_vehicles(dt, &spatial, &snapshot);     // cars only
self.step_motorbikes_sublane(dt, &spatial, &snapshot); // NEW
self.step_pedestrians(dt, &spatial, &snapshot);  // REPLACED
self.detect_gridlock();
self.remove_finished_agents();
self.update_metrics();
```

**6. Modify `step_vehicles()` to accept spatial+snapshot, filter to cars only:**
Change filter from `!= Pedestrian` to `== Car`. Signature becomes `step_vehicles(&mut self, dt: f64, spatial: &SpatialIndex, snapshot: &AgentSnapshot)`. Add cross-type avoidance: when a pedestrian is detected within 15m ahead via spatial query, use that distance as the IDM gap (vehicles brake for jaywalkers per CONTEXT.md).

**7. Create `step_motorbikes_sublane(&mut self, dt, spatial, snapshot)`:**
- Query motorbikes: entities with (RoadPosition, Kinematics, IdmParams, LateralOffset, VehicleType) where vtype == Motorbike
- For each motorbike:
  a. Get road width from edge lane_count * 3.5 (look up edge in road_graph)
  b. Use spatial.nearest_within_radius(pos, 10.0) to find neighbors
  c. Build NeighborInfo vec: for each neighbor, compute longitudinal gap (along heading direction) and lateral offset. Use snapshot to look up neighbor type, speed, and half-width (car=0.9, motorbike=0.25, pedestrian=0.3)
  d. Check if at red light via check_signal_red()
  e. Call sublane::compute_desired_lateral() to get desired lateral
  f. Call sublane::apply_lateral_drift() to get new lateral offset
  g. Run IDM for longitudinal: find leader on same edge (or nearest ahead via spatial), compute idm_acceleration + integrate_with_stopping_guard
  h. Cross-type avoidance: if pedestrian within 3m ahead, treat as IDM leader with speed 0 (force braking). If car within 3m laterally, motorbike yields by swerving (already handled by sublane gap-seeking rejecting occupied gaps)
- Collect updates: (entity, new_speed, new_offset_m, new_lateral, at_red)
- Apply updates: update LateralOffset, RoadPosition.offset_m, handle edge transitions, call update_agent_state_with_lateral for motorbikes

**8. Create `update_agent_state_with_lateral(entity, speed, lateral_offset)`:**
Same as update_agent_state but offsets world position perpendicular to heading:
```rust
let perp_x = -heading.sin();
let perp_y = heading.cos();
pos.x = base_x + lateral_offset * perp_x;
pos.y = base_y + lateral_offset * perp_y;
```

**9. Replace `step_pedestrians()` body:**
Signature: `step_pedestrians(&mut self, dt: f64, spatial: &SpatialIndex, snapshot: &AgentSnapshot)`
- Query pedestrians: (Entity, Position, Kinematics, Route, VehicleType::Pedestrian)
- For each:
  a. Compute destination from next waypoint in route
  b. spatial.nearest_within_radius(pos, 5.0), limit to 10 nearest
  c. Build PedestrianNeighbor vec from ALL agent types in range
  d. Call social_force::social_force_acceleration()
  e. Call social_force::integrate_pedestrian() with max_speed=2.0
  f. Update Position and Kinematics
  g. If within threshold of waypoint, advance route.current_step
- Jaywalking: skip for POC simplicity if it threatens the 700-line limit. The social force model already creates realistic pedestrian movement. If there is room, evaluate should_jaywalk once per sim-second for pedestrians near roads.

**10. Modify `build_instances()` for swarming color and lateral offset:**
Update the ECS query to:
```rust
self.world.query::<(&Position, &Kinematics, &VehicleType, Option<&WaitState>, Option<&LateralOffset>)>()
```
For motorbikes:
- If WaitState.at_red_signal: color = [0.4, 1.0, 0.5, 1.0] (brighter green, swarming)
- Else: color = [0.2, 0.8, 0.4, 1.0] (normal)
- If LateralOffset present: offset position perpendicular to heading (same math as update_agent_state_with_lateral)

Keep sim.rs under 700 lines. If approaching limit, extract AgentSnapshot struct and collect_all_agents into a new `sim_snapshot.rs` module.
