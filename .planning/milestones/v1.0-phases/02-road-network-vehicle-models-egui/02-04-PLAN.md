    **Step 2: Rewrite GpuState in app.rs**

    This is a substantial rewrite of `crates/velos-gpu/src/app.rs`. The new GpuState holds:
    - All existing fields (window, surface, device, queue, world, buffer_pool, dispatcher, renderer, camera)
    - `road_graph: RoadGraph` -- loaded from District 1 PBF at startup
    - `spatial_index: SpatialIndex` -- rebuilt each frame
    - `spawner: Spawner` -- OD + ToD demand spawner
    - `signal_controllers: Vec<FixedTimeController>` -- one per signalized intersection
    - `gridlock_detector: GridlockDetector`
    - `sim_time: f64` -- simulation clock in seconds
    - `sim_state: SimState` -- Running / Paused / Stopped
    - `speed_mult: f32` -- speed multiplier (0.1x to 4.0x) for egui slider
    - `metrics: SimMetrics` -- frame_time_ms, agent_count, motorbike_count, car_count, ped_count, agents_per_sec
    - `egui_state: egui_winit::State`
    - `egui_renderer: egui_wgpu::Renderer`

    **SimState enum:** `Stopped, Running, Paused` with methods `is_running()`, `is_paused()`.

    **SimMetrics struct:** `frame_time_ms: f64, agent_count: u32, motorbike_count: u32, car_count: u32, ped_count: u32, agents_per_sec: f64`.

    **GpuState::new() changes:**
    1. Load road graph: `import_osm("data/hcmc/district1.osm.pbf", 10.7756, 106.7019)`
    2. Create spawner: `Spawner::new(OdMatrix::district1_poc(), TodProfile::hcmc_weekday(), 42)`
    3. Create signal controllers for major intersections (identify from road graph nodes with degree >= 4). Create a default 2-phase signal plan (30s/3s green/amber per phase) for each.
    4. Create gridlock detector with 300s timeout.
    5. Initialize egui: `egui_winit::State::new(...)` and `egui_wgpu::Renderer::new(...)`.
    6. Start with SimState::Stopped, speed_mult=1.0.
    7. Remove the hardcoded 1K circular orbit agent setup. World starts empty; agents come from spawner.
    8. Camera centers on District 1 road network bounding box.

    **GpuState::update() changes:**

    The new update loop (runs only when SimState::Running):
    1. Compute effective dt = base_dt * speed_mult
    2. Advance sim_time += effective_dt
    3. Spawn agents: call `spawner.generate_spawns(sim_time / 3600.0, effective_dt)`. For each SpawnRequest:
       - Pick a random node in the origin zone as start position
       - Pick a random node in the destination zone as target
       - Find route via `find_route`
       - Spawn hecs entity with Position, Kinematics, VehicleType, RoadPosition, Route, WaitState
       - Assign IDM params based on vehicle type
    4. Build spatial index from all agent positions
    5. For each car/motorbike agent:
       - Find leader on same edge/lane via spatial index
       - Compute IDM acceleration
       - Integrate with stopping guard
       - Update position along edge (advance offset_m, handle edge transitions when reaching end)
       - Periodically (every ~0.5s) evaluate MOBIL lane change for cars on multi-lane edges
    6. For each pedestrian: linear interpolation toward destination
    7. Tick signal controllers
    8. Check if agents at intersection nodes should stop (signal red)
    9. Build gridlock waiting graph (agent -> leader mapping for stopped agents not at red signal)
    10. Run gridlock detection. For detected cycles, teleport one agent per cycle to next route node.
    11. Update world positions from edge-local to world coordinates
    12. Remove agents that reached their destination
    13. Build per-type instance arrays and call `renderer.update_instances_typed()`
    14. Update metrics

    NOTE: This is a lot of logic. Keep it organized with helper methods: `spawn_agents()`, `step_vehicles()`, `step_pedestrians()`, `step_signals()`, `detect_gridlock()`, `build_instances()`. If app.rs exceeds 700 lines, extract simulation logic into a separate `sim.rs` module within velos-gpu.

    **GpuState::render() changes:**
    After the existing render pass, add egui render pass. Do NOT remove the compute dispatch -- keep it for now even if IDM runs on CPU. The compute shader can be a no-op or removed if not needed.

    **egui integration:**
    1. In `GpuState::new()`: initialize egui_winit::State and egui_wgpu::Renderer per the research pattern.
    2. In `window_event()`: pass events to `egui_state.on_window_event()` FIRST. Check `egui_ctx.wants_pointer_input()` / `wants_keyboard_input()` before forwarding to camera (prevents egui stealing camera events per pitfall 5).
    3. In render(): after simulation render pass, run egui:
       ```
       let raw_input = egui_state.take_egui_input(&window);
       let full_output = egui_ctx.run(raw_input, |ctx| {
           egui::SidePanel::left("controls").exact_width(240.0).show(ctx, |ui| {
               ui.heading("VELOS");
               ui.separator();
               // Controls (APP-01)
               ui.heading("Controls");
               if ui.button(if running { "Pause" } else { "Start" }).clicked() { toggle }
               if ui.button("Reset").clicked() { reset sim }
               ui.add(egui::Slider::new(&mut speed_mult, 0.1..=4.0).text("Speed"));
               ui.separator();
               // Metrics (APP-02)
               ui.heading("Metrics");
               ui.label(format!("Frame: {:.1}ms", metrics.frame_time_ms));
               ui.label(format!("Agents: {}", metrics.agent_count));
               ui.label(format!("  Motorbikes: {}", metrics.motorbike_count));
               ui.label(format!("  Cars: {}", metrics.car_count));
               ui.label(format!("  Pedestrians: {}", metrics.ped_count));
               ui.label(format!("Throughput: {:.0}/s", metrics.agents_per_sec));
           });
       });
       // Render egui output as second render pass
       ```
    4. Handle egui texture updates and paint jobs via egui_wgpu::Renderer.

    **Agent-to-world coordinate mapping (NET-04):**
    Each agent's Position (x, y) in ECS is in world metres (same coordinate system as the road graph, which was projected at import time). The renderer already works in metres. So: agent offset_m along an edge -> interpolate edge geometry polyline -> get (x, y) in metres -> that IS the Position. No separate transform needed -- NET-04 is satisfied by the equirectangular projection done at import time.

    - Cars: blue [0.2, 0.4, 0.9, 1.0] (new rectangles)
    - Pedestrians: white [0.9, 0.9, 0.9, 1.0] (new dots)
