Create `crates/velos-net/src/graph.rs`: `RoadClass` enum (Primary, Secondary, Tertiary, Residential). `RoadNode { pos: [f64; 2] }`. `RoadEdge { length_m, speed_limit_mps, lane_count: u8, oneway: bool, road_class, geometry: Vec<[f64; 2]> }`. `RoadGraph` wrapping `petgraph::graph::DiGraph<RoadNode, RoadEdge>` with accessors: `node_count()`, `edge_count()`, `inner()` (returns &DiGraph), `node_position(NodeIndex) -> [f64; 2]`.

       - Pass 1: `ElementReader::from_path`, iterate elements, collect node coordinates (HashMap<i64, (f64, f64)>) and highway ways with their node refs + tags.
       - Pass 2: For each highway way, parse tags (road class, lanes, speed limit, oneway). Skip non-matching highway types. Project all node coordinates to local metres. Build DiGraph edges: for each consecutive node pair in the way, create a directed edge. If not oneway, also create reverse edge. Compute edge length from projected coordinates.
       - Tag parsing: `parse_road_tags` function per research pattern. `infer_lanes(RoadClass)`: Primary/Secondary -> 2, Tertiary/Residential -> 1. Default speed: Primary -> 50 km/h, Secondary -> 40, Tertiary -> 30, Residential -> 20.

       - `from_positions(ids: &[u32], positions: &[[f64; 2]]) -> Self` using `RTree::bulk_load`
       - `nearest_within_radius(pos: [f64; 2], radius: f64) -> Vec<&AgentPoint>` using `locate_within_distance`
       - `nearest_neighbor(pos: [f64; 2]) -> Option<&AgentPoint>` using `nearest_neighbor`
       Rebuild the entire R-tree each frame via `from_positions` (bulk_load is O(n log n), better than incremental insert/remove per the research anti-pattern guidance).

       - Heuristic = Euclidean distance between node positions / max_speed (admissible heuristic)
