  <name>Task 1: velos-vehicle crate -- IDM + MOBIL + VehicleType + gridlock</name>
  <files>

  <behavior>
    IDM:
    - Free-flow (no leader): acceleration approaches a_max when gap is very large
    - Following close leader: acceleration is negative (braking)
    - Zero speed with leader: v_eff kickstart (0.1 m/s) prevents division issues
    - Ballistic stopping guard: v + a*dt < 0 results in v_new=0 and dx >= 0 (no negative velocity)
    - Acceleration clamped to [-9.0, a_max] range
    - Approaching stopped leader: deceleration increases as gap shrinks

    MOBIL:
    - Safety criterion: reject if new follower deceleration < safe_decel (-4.0)
    - Incentive: accept when own_advantage - politeness*follower_disadvantage + bias > threshold
    - Right bias: positive for right lane change, negative for left
    - Politeness=0.3: moderate altruism (HCMC default)

    Gridlock:
    - Cycle in waiting graph (A->B->C->A) detected as gridlock
    - Linear chain (A->B->C with no cycle) not flagged
    - Multiple separate cycles detected independently
    - Empty waiting graph returns no cycles
    - Agents waiting at red signal excluded from gridlock consideration (caller responsibility, but tested via documentation)

    VehicleType:
    - Enum: Motorbike, Car, Pedestrian
    - default_idm_params for each type (Car: v0=13.9 m/s, s0=2.0, T=1.5, a=1.0, b=2.0; Motorbike: v0=11.1, s0=1.0, T=1.0, a=2.0, b=3.0)
  </behavior>
  <action>
    1. Add `"crates/velos-vehicle"` to workspace members in root Cargo.toml.

    2. Create `crates/velos-vehicle/Cargo.toml`: dependencies on thiserror (workspace), log (workspace). No external deps needed -- IDM/MOBIL are pure math.

    3. Create `crates/velos-vehicle/src/error.rs`: `VehicleError` enum.

    4. Create `crates/velos-vehicle/src/types.rs`:
       - `VehicleType` enum: `Motorbike, Car, Pedestrian` with `#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]`
       - `IdmParams { v0: f64, s0: f64, t_headway: f64, a: f64, b: f64, delta: f64 }`
       - `MobilParams { politeness: f64, threshold: f64, safe_decel: f64, right_bias: f64 }`
       - `default_idm_params(VehicleType) -> IdmParams`:
         Car: v0=13.9 (50 km/h), s0=2.0, T=1.5, a=1.0, b=2.0, delta=4.0
         Motorbike: v0=11.1 (40 km/h), s0=1.0, T=1.0, a=2.0, b=3.0, delta=4.0
         Pedestrian: v0=1.4 (5 km/h), s0=0.5, T=0.5, a=0.5, b=1.0, delta=4.0
       - `default_mobil_params() -> MobilParams`: politeness=0.3, threshold=0.2, safe_decel=-4.0, right_bias=0.1

    5. Create `crates/velos-vehicle/src/idm.rs` following the research code pattern exactly:
       - `idm_acceleration(params, v, gap, delta_v) -> f64`: free-road term (1 - (v/v0)^4), desired gap s*, interaction term (s*/gap)^2, clamp to [-9.0, a_max]. Use v_eff = v.max(0.1) for zero-speed kickstart. Use gap_eff = gap.max(0.01) to avoid division by zero.
       - `integrate_with_stopping_guard(v, accel, dt) -> (f64, f64)`: returns (v_new, dx). If v + a*dt < 0, compute t_stop = (-v/accel).min(dt), integrate only to t_stop, return (0.0, dx.max(0.0)).

    6. Create `crates/velos-vehicle/src/mobil.rs` following the research code pattern:
       - `LaneChangeContext { accel_current, accel_target, accel_new_follower, accel_old_follower, is_right }`
       - `mobil_decision(params, ctx) -> bool`: safety check (new_follower >= safe_decel), then incentive = own_advantage - politeness * follower_disadvantage + bias > threshold.

    7. Create `crates/velos-vehicle/src/gridlock.rs`:
       - `GridlockDetector { timeout_secs: f64 }` with `new(timeout_secs)` defaulting to 300.0.
       - `detect_cycles(waiting_graph: &HashMap<u32, u32>) -> Vec<Vec<u32>>`: BFS cycle detection per research pattern. For each unvisited start node, follow the chain. If we revisit a node in the current path, extract the cycle. Skip nodes already visited by previous iterations.
       - Use `u32` for agent IDs (consistent with spatial index).

    8. Create `crates/velos-vehicle/src/lib.rs`: re-export all modules.

    9. Write TDD tests (RED then GREEN):
       - `tests/idm_tests.rs`: free-flow acceleration, following deceleration, zero-speed kickstart, stopping guard prevents negative velocity, clamp range.
       - `tests/mobil_tests.rs`: safety rejection, incentive acceptance, incentive rejection below threshold, right bias effect.
       - `tests/gridlock_tests.rs`: cycle detection (3-node cycle), no false positive on linear chain, multiple cycles, empty graph.

  <name>Task 2: velos-signal crate -- fixed-time traffic signal controller</name>
  <files>

  <behavior>
    - SignalPlan holds a list of phases with green/amber/red durations
    - FixedTimeController ticks sim_time and cycles through phases
    - At t=0, first phase is green
    - After green_duration, phase transitions to amber
    - After amber_duration, phase transitions to red and next phase starts green
    - Full cycle wraps around to first phase
    - get_phase_state(approach_index) returns current PhaseState (Green/Amber/Red) for that approach
    - Multiple controllers (one per intersection) tick independently
  </behavior>
  <action>
    1. Add `"crates/velos-signal"` to workspace members in root Cargo.toml.

    2. Create `crates/velos-signal/Cargo.toml`: dependencies on thiserror (workspace), log (workspace).

    3. Create `crates/velos-signal/src/error.rs`: `SignalError` enum.

    4. Create `crates/velos-signal/src/plan.rs`:
       - `PhaseState` enum: `Green, Amber, Red` with `#[derive(Debug, Clone, Copy, PartialEq, Eq)]`
       - `SignalPhase { green_duration: f64, amber_duration: f64, approaches: Vec<usize> }` -- which intersection approaches get green during this phase
       - `SignalPlan { phases: Vec<SignalPhase>, cycle_time: f64 }` with `new(phases)` that computes cycle_time = sum of (green + amber) for all phases + red gap. Each phase's total duration = green + amber. All phases sum to cycle_time.

    5. Create `crates/velos-signal/src/controller.rs`:
       - `FixedTimeController { plan: SignalPlan, elapsed: f64, num_approaches: usize }`
       - `new(plan, num_approaches) -> Self` with elapsed=0.0
       - `tick(dt: f64)`: advance elapsed by dt, wrap around cycle_time
       - `get_phase_state(approach_index: usize) -> PhaseState`: determine which phase is active at current elapsed time, check if this approach is in the active phase's approaches list. If yes, return Green or Amber based on time within phase. If no, return Red.
       - `reset()`: set elapsed=0.0

    6. Create `crates/velos-signal/src/lib.rs`: re-export plan and controller.

    7. Write TDD tests in `tests/signal_tests.rs`:
       - Simple 2-phase plan (NS green 30s + 3s amber, EW green 25s + 3s amber)
       - At t=0: NS approach is Green, EW approach is Red
       - At t=30: NS is Amber, EW is Red
       - At t=33: NS is Red, EW is Green
       - At t=58: EW is Amber
       - At t=61: wraps to t=0, NS is Green again
       - Reset sets back to initial state
  </action>
