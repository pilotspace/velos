    - generate_spawns for 1 hour at peak (factor=1.0) produces approximately OD matrix total trips
    - generate_spawns at off-peak (factor=0.1) produces ~10% of peak volume
    - Over 1000+ spawns, vehicle type distribution converges to 80% motorbike, 15% car, 5% pedestrian (within 5% tolerance)
    - generate_spawns with empty OD matrix produces no spawns
    - Each spawn has a valid origin and destination zone (not equal)

  <action>
    1. Create `crates/velos-demand/src/spawner.rs`:
       - Import `VehicleType` definition locally (do NOT depend on velos-vehicle crate -- define a local `SpawnVehicleType` enum or use a string/u8 tag). Actually, to keep it simple and avoid duplication, use a `u8` tag: 0=Motorbike, 1=Car, 2=Pedestrian, with constants. The integration plan (02-04) will map to the real VehicleType enum.

       Wait -- better approach: define `VehicleType` enum right here in velos-demand since it's a data type shared across crates. Actually, the CONTEXT.md says velos-vehicle has the types. To avoid circular deps, define a simple enum here:

       `SpawnVehicleType` enum: `Motorbike, Car, Pedestrian`

       - `SpawnRequest { origin: Zone, destination: Zone, vehicle_type: SpawnVehicleType }`
       - `Spawner { od: OdMatrix, tod: TodProfile, rng: StdRng }`
       - `new(od, tod, seed: u64) -> Self`: create spawner with seeded RNG for reproducibility
       - `generate_spawns(sim_hour: f64, dt: f64) -> Vec<SpawnRequest>`:
         1. Get ToD factor for current hour
         2. For each OD pair, compute expected spawns = trips_per_hour * factor * (dt / 3600.0)
         3. Use Poisson-like approach: if expected < 1.0, spawn with probability = expected. If expected >= 1.0, spawn floor(expected) agents + 1 more with probability = fractional part.
         4. For each spawn, assign vehicle type via weighted random: 80% Motorbike, 15% Car, 5% Pedestrian using rand::distributions::WeightedIndex.
         5. Return all spawn requests.

    2. Update `crates/velos-demand/src/lib.rs` to export spawner module.

       - Spawner tests: spawn count proportional to ToD factor, vehicle type distribution over 10000 spawns converges to 80/15/5 (within 5% tolerance), empty OD produces no spawns, seeded RNG produces deterministic output
