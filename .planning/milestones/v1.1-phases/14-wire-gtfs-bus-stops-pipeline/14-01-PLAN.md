<objective>
Create the two foundation modules for the GTFS-to-bus-stops pipeline: (1) edge segment R-tree snapping in velos-net that converts GtfsStop lat/lon coordinates to BusStop edge_id + offset_m, and (2) BusSpawner in velos-demand that time-gates bus agent spawning by GTFS trip departure schedules.

Purpose: These modules bridge the GTFS parser output (lat/lon stops) to the simulation's edge-based bus stop format, enabling Plan 02 to wire them into SimWorld.
Output: Two new modules with unit tests -- snap.rs in velos-net, bus_spawner.rs in velos-demand.
</objective>

    - Test: snap_gtfs_stops converts Vec of GtfsStop into Vec of BusStop, projecting WGS84 lat/lon to local metres via EquirectangularProjection, skipping stops beyond 50m

    - Test: snap_gtfs_stops with zero stops returns empty Vec (graceful empty input)

  <action>
Create `crates/velos-net/src/snap.rs` with:

1. `EdgeSegment` struct implementing `rstar::RTreeObject` with AABB envelope. Fields: `edge_id: u32`, `segment_start: [f64; 2]`, `segment_end: [f64; 2]`, `offset_along_edge: f64` (cumulative distance from edge start to segment start).

2. Implement `rstar::PointDistance` for `EdgeSegment` -- compute the actual point-to-segment distance (perpendicular projection, not just AABB distance). This is critical for accurate nearest-neighbor results from rstar.

3. `build_edge_rtree(graph: &RoadGraph) -> RTree<EdgeSegment>` -- iterate all edges via `graph.inner().edge_indices()`, decompose each edge's `geometry` polyline into consecutive segments, compute cumulative offset for each segment, collect into `RTree::bulk_load()`.

4. `project_onto_segment(point: [f64; 2], seg_start: [f64; 2], seg_end: [f64; 2]) -> (f64, [f64; 2], f64)` -- returns (t_param, nearest_point, distance). Clamp t to [0, 1]. Handle degenerate zero-length segments.

5. `snap_to_nearest_edge(tree: &RTree<EdgeSegment>, point: [f64; 2], max_radius: f64) -> Option<(u32, f64, f64)>` -- use `tree.nearest_neighbor()` to find closest segment, compute exact perpendicular projection, return `(edge_id, offset_m, distance)` where `offset_m = segment.offset_along_edge + t * segment_length`. Return None if distance > max_radius.

6. `snap_gtfs_stops(stops: &[GtfsStop], graph: &RoadGraph, proj: &EquirectangularProjection) -> Vec<BusStop>` -- the high-level function. Builds R-tree, projects each GtfsStop lat/lon to local metres via `proj.project(stop.lat, stop.lon)`, snaps to nearest edge, logs warning for skipped stops (>50m), then merges duplicates on same edge within 10m. Returns `Vec<BusStop>` with `capacity: 40` default.

Add `pub mod snap;` and `pub use snap::{snap_gtfs_stops, build_edge_rtree};` to `crates/velos-net/src/lib.rs`.

Note: rstar is already a workspace dependency used by velos-net (spatial.rs). No new dependencies needed.

Create `crates/velos-demand/src/bus_spawner.rs` with:

1. `BusSpawnRequest` struct: `route_id: String`, `trip_id: String`, `stop_indices: Vec<usize>` (indices into the global bus_stops Vec).

2. `BusSpawner` struct:
   - `route_stops: HashMap<String, Vec<usize>>` -- per-route ordered stop indices into bus_stops Vec
   - `schedules: Vec<BusSchedule>` -- sorted by first stop_time departure_s
   - `next_trip_index: usize` -- index of next unspawned trip

3. `BusSpawner::new(routes: &[BusRoute], schedules: Vec<BusSchedule>, bus_stops: &[BusStop]) -> Self`:
   - Build route_stops by matching each route's GtfsStop.stop_id to the bus_stops Vec. For each route, iterate its stops, find the corresponding BusStop index in bus_stops by matching stop name or by re-snapping stop_id. Since snap_gtfs_stops produces BusStops with names from GtfsStop.name, match by name.
   - Actually, a cleaner approach: accept a `stop_id_to_bus_stop_index: &HashMap<String, usize>` parameter that maps GtfsStop.stop_id to index in bus_stops (built during snapping in Plan 02). This decouples snapping from spawner construction.
   - Sort schedules by `stop_times[0].departure_s` (first stop departure = trip start time). Set next_trip_index = 0.

4. `BusSpawner::generate_bus_spawns(&mut self, sim_time_s: f64) -> Vec<BusSpawnRequest>`:
   - Convert sim_time_s to seconds-from-midnight (sim_time_s % 86400 for time-of-day matching).
   - While `next_trip_index < schedules.len()` and `schedules[next_trip_index].first_departure_s <= sim_time_sod`: emit BusSpawnRequest with route_stops lookup for stop_indices, advance next_trip_index.
   - Return Vec of ready spawn requests.

Add `pub mod bus_spawner;` and `pub use bus_spawner::{BusSpawner, BusSpawnRequest};` to `crates/velos-demand/src/lib.rs`.

Note: BusSpawner does NOT depend on velos-net (no CCH path computation here). Route edge paths will be computed at spawn time in Plan 02 using the existing Route/CCH infrastructure.
