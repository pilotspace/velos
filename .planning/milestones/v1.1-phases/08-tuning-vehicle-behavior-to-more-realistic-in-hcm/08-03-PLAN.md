<objective>
Add HCMC-specific behavioral rules: red-light creep for motorbikes, aggressive speed-dependent weaving gap thresholds, and vehicle-type-dependent intersection gap acceptance with size intimidation.

Purpose: These three behaviors are what make HCMC traffic visually distinct from Western traffic models. Red-light creep creates the characteristic motorbike swarm at intersections. Aggressive weaving lets motorbikes filter through traffic realistically. Yield-based intersection negotiation replaces rigid priority rules with the "organized chaos" seen at HCMC unsignalized intersections.

Output: intersection.rs module, updated sublane.rs with creep and speed-dependent gaps, config params for all new behaviors.
</objective>

       ```rust
       /// Determine if a vehicle should proceed through an unsignalized intersection.
       ///
       /// Uses vehicle-type-dependent TTC thresholds modified by:
       /// - Size intimidation: larger approaching vehicles increase required TTC
       /// - First-come priority: longer wait reduces effective threshold
       /// - Max-wait forced acceptance: after 5s, threshold halved to break deadlock
       ///
       /// # Arguments
       /// * `own_type` - vehicle type of the deciding agent
       /// * `other_type` - vehicle type of the approaching/conflicting agent
       /// * `ttc` - time-to-collision with the conflicting agent (s)
       /// * `own_ttc_threshold` - base gap acceptance threshold from config (s)
       /// * `state` - intersection waiting state
       ///
       /// # Returns
       /// `true` if the vehicle should proceed (gap is acceptable)
       pub fn intersection_gap_acceptance(
           own_type: VehicleType,
           other_type: VehicleType,
           ttc: f64,
           own_ttc_threshold: f64,
           state: &IntersectionState,
       ) -> bool
       ```

       - Car approaching: factor = 1.0
