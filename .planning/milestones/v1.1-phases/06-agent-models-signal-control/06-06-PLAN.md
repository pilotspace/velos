  <action>
    1. Create crates/velos-gpu/shaders/pedestrian_adaptive.wgsl with 4 entry points (4 dispatches):

       **Pass 1 - Count:** `@compute @workgroup_size(256) fn count_per_cell`
       - Each pedestrian hashes its position to a spatial cell: cell_x = floor(pos_x / cell_size), cell_y = floor(pos_y / cell_size)
       - Cell size is a uniform param (2.0, 5.0, or 10.0m based on density classification)
       - Use `atomicAdd(&cell_counts[cell_id], 1u)` to count pedestrians per cell
       - Store cell assignment per pedestrian in a mapping buffer

       **Pass 2 - Prefix Sum:** `@compute @workgroup_size(256) fn prefix_sum`
       - Hillis-Steele scan within each workgroup (shared memory)
       - For multi-workgroup: two-pass reduce-then-scan
         - First dispatch: per-workgroup prefix sums + store workgroup totals
         - Second dispatch: scan workgroup totals, add back to each element
       - For VELOS ~20K pedestrians with 2m cells, expect ~5K-10K non-empty cells. Single workgroup scan may NOT suffice. Implement the multi-workgroup path.
       - Output: cell_offsets array (exclusive prefix sum of cell_counts)

       **Pass 3 - Scatter:** `@compute @workgroup_size(256) fn scatter`
       - Each pedestrian writes its index to compacted_indices[cell_offsets[cell_id] + local_offset]
       - local_offset obtained via atomicAdd on a per-cell counter (separate from cell_counts)

       **Pass 4 - Social Force:** `@compute @workgroup_size(64) fn social_force_adaptive`
       - One workgroup per non-empty cell (indirect dispatch using compacted cell count)
       - Thread processes one pedestrian from compacted_indices
       - Neighbor search only within same and adjacent cells (9-cell neighborhood)
       - Implements social force model matching CPU reference:
         - Desired force: (v0 * e_desired - v_current) / tau
         - Social repulsion: a_social * exp(-distance / b_social) * direction
         - Anisotropic weighting: cos(angle) factor from CPU reference

    2. In compute.rs, add `dispatch_pedestrian_adaptive`:
       - Create 4 compute pipelines (count, prefix_sum, scatter, social_force)
       - Create GPU buffers: cell_counts, cell_offsets, compacted_indices, pedestrian_cell_map
       - Orchestrate 4 dispatches in sequence with appropriate barriers
       - Use indirect dispatch for Pass 4 (workgroup count = number of non-empty cells)
       - Density classification: count total pedestrians / area to determine cell_size uniform

    IMPORTANT: Follow anti-patterns from RESEARCH.md:
    - DO NOT use dynamic @workgroup_size (WGSL requires compile-time constants)
    - Use multiple dispatches, NOT cross-workgroup atomics for prefix sum
    - Workgroup size 256 for compute passes, 64 for social force (matches architecture doc)
