    - glosa_speed returns max_speed if already green (time_to_green == 0)
    - glosa_speed returns 0 if cannot make it at max_speed
    - PriorityRequest: approach_index, level (Emergency > Bus), vehicle_id
    - PriorityQueue: accepts requests, dequeues highest priority, max 1 per cycle
    - Priority green extension: extend current green by up to 15s if approach is green
    - Priority red shortening: reduce conflicting green by up to 10s if approach is red
    - Emergency priority always overrides bus priority

    1. Create crates/velos-signal/src/spat.rs:
       - `SpatBroadcast` struct: approach_states: Vec<PhaseState>, time_to_next_change: f64, cycle_time: f64
       - `glosa_speed(distance_m: f64, time_to_green_s: f64, v_max: f64) -> f64`:
         - If time_to_green == 0: return v_max (already green)
         - Compute required_speed = distance_m / time_to_green_s
         - If required_speed > v_max: return 0.0 (can't make it, stop)
         - If required_speed < 3.0 (too slow): return 0.0
         - Else return required_speed (approach at this speed to arrive on green)

       - `PriorityRequest` struct: approach_index: usize, level: PriorityLevel, vehicle_id: u32
       - `PriorityQueue` struct: requests: Vec<PriorityRequest>, served_this_cycle: bool
       - `PriorityQueue::submit(request: PriorityRequest)` -- adds to queue
       - `PriorityQueue::dequeue() -> Option<PriorityRequest>` -- returns highest priority if not yet served this cycle; sets served_this_cycle=true
       - `PriorityQueue::reset_cycle()` -- clears served flag
       - `apply_priority(controller: &mut dyn SignalController, request: &PriorityRequest)` -- extends green by up to 15s or shortens conflicting red by up to 10s

    3. Extend SignalController trait with:
       - `fn spat_data(&self) -> SpatBroadcast` -- default implementation computes from get_phase_state
       - `fn request_priority(&mut self, request: PriorityRequest)` -- default implementation is no-op
