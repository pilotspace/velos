  <behavior>
    - interpolate_idm_params at distance 0m (meso boundary) returns relaxed params (t_headway=3.0, a=0.5)
    - interpolate_idm_params at distance 100m (micro boundary) returns normal params (original values)
    - interpolate_idm_params at distance 50m returns smooth-step interpolated values
    - Smooth-step: t = 3x^2 - 2x^3 where x = distance / 100m (C1-continuous)
    - velocity_matching_speed(meso_exit_speed, last_micro_speed) returns min of both
    - BufferZone::should_insert returns true when buffer distance and speed conditions are met
  </behavior>
  <action>
    1. Create crates/velos-meso/src/buffer_zone.rs:
       - `BufferZone` struct: buffer_length: f64 (100.0m default), relaxed_params: IdmParams, normal_params: IdmParams
       - `smoothstep(x: f64) -> f64` -- 3x^2 - 2x^3, clamped to [0,1] (per RESEARCH.md recommendation for C1 continuity)
       - `interpolate_idm_params(relaxed: &IdmParams, normal: &IdmParams, distance_into_buffer: f64, buffer_length: f64) -> IdmParams`:
         - x = (distance_into_buffer / buffer_length).clamp(0.0, 1.0)
         - t = smoothstep(x)
         - Each param: relaxed_val + (normal_val - relaxed_val) * t
       - Relaxed params: t_headway=3.0 (more lenient), a=0.5 (gentler accel), b=1.5, v0=same as normal, s0=same
       - `velocity_matching_speed(meso_exit_speed: f64, last_micro_vehicle_speed: f64) -> f64`:
         - Returns min(meso_exit_speed, last_micro_vehicle_speed)
         - This prevents speed mismatch at boundary per RESEARCH.md Pitfall 4
       - `BufferZone::should_insert(distance_into_buffer: f64, speed_diff: f64) -> bool`:
         - Agent can enter micro zone when distance >= buffer_length AND speed difference < 2.0 m/s

       - Interpolation at boundaries (0m and 100m)
       - Interpolation at 50m is between relaxed and normal
       - C1 continuity: derivative at boundaries is ~0 (verify with finite differences)
       - velocity_matching_speed takes minimum
       - should_insert conditions
