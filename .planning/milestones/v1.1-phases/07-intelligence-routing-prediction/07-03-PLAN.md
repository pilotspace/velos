    Create `crates/velos-net/src/cch/customization.rs` implementing the bottom-up triangle enumeration algorithm from the research doc:

    ```rust
    impl CCHRouter {
        pub fn customize(&mut self, original_weights: &[f32]) {
            // 1. Initialize CCH edges from original graph weights
            // 2. Process nodes bottom-up by rank
            // 3. For each node, enumerate (down-edge, up-edge) pairs
            // 4. Update shortcut weight = min(current, down_weight + up_weight)
        }
    }
    ```

    Key implementation details:
    - Initialize: for each original edge, set `forward_weight[cch_idx] = original_weights[orig_idx]` and same for backward
    - Bottom-up: iterate ranks 0..node_count
    - For each rank's node, scan backward_first_out[node]..backward_first_out[node+1] (incoming down-edges) and forward_first_out[node]..forward_first_out[node+1] (outgoing up-edges)
    - For each (down, up) pair, find the direct edge between their endpoints and update with min
    - Use a temporary HashMap or sorted lookup for edge finding between arbitrary node pairs

    Also add `pub fn customize_with_fn(&mut self, weight_fn: impl Fn(usize) -> f32)` that takes a closure mapping original edge index to weight (convenient for prediction overlay integration).

    Add `pub mod customization;` to cch/mod.rs.

  <action>
    Create `crates/velos-net/src/cch/query.rs`:

    1. `pub fn query(&self, source: u32, target: u32) -> Option<f32>`:
       - Bidirectional Dijkstra on upper graph (upward edges only)
       - Forward search from source using forward_star, exploring nodes with increasing rank
       - Backward search from target using backward_star, exploring nodes with increasing rank
       - Alternate between forward and backward expansions
       - Track best meeting point: for each node settled by both searches, total_cost = forward_dist[node] + backward_dist[node]
       - Return minimum total_cost across all meeting points
       - Use `std::collections::BinaryHeap` with `Reverse` for min-heap

    2. `pub fn query_with_path(&self, source: u32, target: u32) -> Option<(f32, Vec<u32>)>`:
       - Same as query but also tracks parent pointers
       - After finding meeting point, reconstruct path by following parents forward + backward
       - Unpack shortcuts recursively: if shortcut_middle[edge] is Some(mid), expand into (source, mid) + (mid, target)
       - Return path as Vec<u32> of node indices

    3. `pub fn query_batch(&self, pairs: &[(u32, u32)]) -> Vec<Option<f32>>`:
       - Uses rayon `.par_iter().map()` for parallel queries
       - Each query is independent, perfect parallelism
