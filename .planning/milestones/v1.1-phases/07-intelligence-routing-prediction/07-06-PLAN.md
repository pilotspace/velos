  <action>
    Create `crates/velos-core/src/reroute.rs`:

    1. `RerouteConfig`:
    ```rust
    pub struct RerouteConfig {
        pub batch_size: usize,              // 1000
        pub cooldown_seconds: f64,          // 30.0
        pub cost_delta_threshold: f32,      // 0.30 (30%)
    }
    impl Default for RerouteConfig { /* locked decision values */ }
    ```

    2. `RerouteScheduler`:
    ```rust
    pub struct RerouteScheduler {
        config: RerouteConfig,
        ring_index: usize,          // current position in round-robin
        total_agents: usize,        // updated when population changes
        immediate_queue: VecDeque<u32>,  // priority triggers (agent IDs)
        last_reroute: HashMap<u32, f64>,  // agent_id -> sim_time of last reroute
    }
    ```

    Implement:
    - `new(config: RerouteConfig) -> Self`
    - `set_population(&mut self, count: usize)` -- called when agent count changes
    - `add_immediate_trigger(&mut self, agent_id: u32)` -- push to immediate_queue
    - `next_batch(&mut self, sim_time: f64) -> Vec<u32>` -- returns up to batch_size agent IDs:
      1. Take from immediate_queue first (filter out those on cooldown)
      2. Fill remainder from round-robin ring (filter out those on cooldown)
      3. Advance ring_index
    - `record_reroute(&mut self, agent_id: u32, sim_time: f64)` -- update last_reroute map
    - `is_on_cooldown(&self, agent_id: u32, sim_time: f64) -> bool`

    3. `RerouteResult` enum:
    ```rust
    pub enum RerouteResult {
        NoReroute,
        ShouldReroute {
            new_route: Vec<u32>,  // edge IDs
            cost_delta: f32,       // improvement fraction
        },
    }
    ```

    4. `evaluate_reroute` function:
    ```rust
    pub fn evaluate_reroute(
        current_route: &[u32],
        current_edge_index: usize,
        agent_destination: u32,
        perception: &PerceptionResult,
        profile_weights: &CostWeights,
        edge_attrs: &[EdgeAttributes],
        overlay: &PredictionOverlay,
        cch: &CCHRouter,
        config: &RerouteConfig,
    ) -> RerouteResult
    ```
    Algorithm:
    1. Compute current_cost = route_cost on remaining edges of current route
    2. If perception.flags bit0 (route_blocked): immediately flag for reroute
    3. Query CCH for alternative path from current position to destination
    4. Compute alt_cost = route_cost on alternative path
    5. cost_delta = (current_cost - alt_cost) / current_cost
    6. If cost_delta > threshold: return ShouldReroute with new_route
    7. Else: return NoReroute

    Add `pub mod reroute;` to velos-core/src/lib.rs.

    Note: This function takes typed parameters, not raw ECS world. The sim.rs integration (wiring into the frame loop) requires reading SUMMARYs from prior plans to understand the sim step structure. For now, reroute.rs is a pure-logic module tested independently.

    Write comprehensive unit tests covering all behaviors listed above. Use mock data (hand-crafted routes, fake PerceptionResult, simple CostWeights).
