In `update_signal_buffer()`, add early return:

Wait -- `update_signal_buffer` takes `&self` currently, not `&mut self`. It's called from `step_perception` which takes `&mut self`. Two options:
(a) Change `update_signal_buffer` to `&mut self` to reset the flag.
(b) Use a Cell<bool> for dirty flags.

Option (a) is simpler. Change the method signatures from `&self` to `&mut self`:
```rust
fn update_signal_buffer(&mut self, queue: &wgpu::Queue) {

    // ... existing upload ...
    self.signal_dirty = false;
}

fn update_edge_travel_ratio_buffer(&mut self, queue: &wgpu::Queue) {
    if !self.prediction_dirty { return; }
    // ... existing upload ...

}
```

These are `&self` currently -- verify call chain allows `&mut self`. Since `step_perception` already takes `&mut self`, this should work. Check and fix any borrow issues.
