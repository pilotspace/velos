<objective>
Wire the PerceptionPipeline's result buffer to the ComputeDispatcher's binding(8) so that
wave_front.wgsl behavior functions (red_light_creep_speed, intersection_gap_acceptance) read
real perception data instead of zeroes.

Purpose: Close the perception buffer integration gap -- the perception shader writes correct
data but it never reaches wave_front because two separate buffers exist.

Output: Single shared buffer created externally, passed to both PerceptionPipeline and
ComputeDispatcher. Integration test proving non-zero perception data flows through.
</objective>

Buffer usage requirements:
- PerceptionPipeline writes via shader (storage read_write) + copies to staging (COPY_SRC)
- ComputeDispatcher reads via shader (storage read) in wave_front binding(8)
- Combined flags needed: STORAGE | COPY_SRC (COPY_SRC covers both needs)

1. In `perception.rs`, add a constructor `new_with_result_buffer(device, max_agents, result_buffer: wgpu::Buffer) -> Self` that:
   - Accepts an externally-created result buffer instead of creating its own
   - Creates the staging buffer with matching size (`max_agents * 32` bytes, `MAP_READ | COPY_DST`)
   - Otherwise identical to `new()` (same pipeline, bind group layout, params buffer)
   - Keep existing `new()` unchanged for backward compatibility

2. In `sim.rs::SimWorld::new()`, after line 193 where `PerceptionPipeline::new()` is called, restructure to:
   ```rust
   // Create the shared perception result buffer with correct usage flags.
   let perc_result_size = (300_000u64) * 32;
   let perc_result_buffer = device.create_buffer(&wgpu::BufferDescriptor {
       label: Some("perception_results_shared"),
       size: perc_result_size,
       usage: wgpu::BufferUsages::STORAGE | wgpu::BufferUsages::COPY_SRC,
       mapped_at_creation: false,
   });

   // Give the buffer to ComputeDispatcher (transfers ownership).
   dispatcher.set_perception_result_buffer(perc_result_buffer);

   // Create PerceptionPipeline using the SAME buffer (via reference from dispatcher).
   // But set_perception_result_buffer takes ownership...
   ```

   IMPORTANT: `wgpu::Buffer` is NOT Clone. The ownership pattern must be:
   - Create the result buffer once
   - Pass ownership to PerceptionPipeline via `new_with_result_buffer()`
   - PerceptionPipeline owns the buffer, ComputeDispatcher gets a reference

   Actually, the SIMPLEST approach: keep PerceptionPipeline as the owner (it needs COPY_SRC for readback). After creating it, take a reference and create the ComputeDispatcher's bind group from that reference. BUT `set_perception_result_buffer()` takes owned `wgpu::Buffer`.

   RECOMMENDED approach: modify `set_perception_result_buffer()` signature is not needed -- instead, DON'T use it. Instead:
   - Create the perception result buffer externally in `SimWorld::new()`
   - Create `PerceptionPipeline` with `new_with_result_buffer()` passing the buffer
   - Wait, that also transfers ownership to PerceptionPipeline...

   ACTUAL simplest approach: The ComputeDispatcher creates its bind group lazily each frame in `dispatch_wave_front()`. It uses `self.perception_result_buffer` to build the bind group entry. So we need ComputeDispatcher to hold the buffer. But PerceptionPipeline also needs it for `readback_results()` (copies from result_buffer to staging).

   The real fix: Create buffer externally. Give ownership to ComputeDispatcher via `set_perception_result_buffer()`. Modify PerceptionPipeline to accept `&wgpu::Buffer` reference for its bind group and readback. But PerceptionPipeline stores `result_buffer: wgpu::Buffer` (owned).

   CLEANEST FIX: Change PerceptionPipeline to NOT own the result buffer. Instead:
   - Remove `result_buffer` field from `PerceptionPipeline`
   - `dispatch()` and `readback_results()` take `&wgpu::Buffer` as parameter
   - The buffer lives in ComputeDispatcher

   This is the most wgpu-idiomatic pattern: one owner, references passed during dispatch.

   IMPLEMENTATION STEPS:
   a. In `perception.rs`:
      - Remove `result_buffer` field from `PerceptionPipeline` struct
      - Remove `result_buffer()` getter
      - Update `new()` to NOT create result_buffer (keep staging_buffer, params_buffer)
      - Update `create_bind_group()` to accept result_buffer as part of `PerceptionBindings` (add a `result_buffer: &'a wgpu::Buffer` field to `PerceptionBindings`)
      - Update `dispatch()` to take `result_buffer: &wgpu::Buffer` parameter
      - Update `readback_results()` to take `result_buffer: &wgpu::Buffer` parameter
      - Staging buffer size: compute from `max_agents * 32` (keep max_agents field)

   b. In `sim.rs::SimWorld::new()`:
      - Create the shared result buffer with `STORAGE | COPY_SRC` usage flags
      - Call `dispatcher.set_perception_result_buffer(result_buffer)` -- ComputeDispatcher owns it
      - Create `PerceptionPipeline::new(device, 300_000)` (no longer needs result buffer)

   c. In `sim_perception.rs` (where `step_perception()` calls dispatch/readback):
      - Pass `dispatcher.perception_result_buffer()` to `perception.dispatch()` and `perception.readback_results()`

   d. Update all call sites of `perception.result_buffer()` to use `dispatcher.perception_result_buffer()` instead.
