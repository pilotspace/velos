  <files>crates/velos-net/src/sumo_import.rs, crates/velos-net/src/lib.rs, crates/velos-net/Cargo.toml, tests/fixtures/simple.net.xml, crates/velos-net/tests/sumo_net_import.rs</files>
  <behavior>
    - Fixture .net.xml with 4 edges, 3 junctions, 4 connections -> RoadGraph with 3 nodes, 4 edges
    - Internal edges (id starting with ":") are filtered out
    - Edge lane count parsed from <lane> child elements
    - Edge speed_limit taken from highest lane speed
    - Junction positions parsed into RoadNode x,y
    - Connection elements create graph edges between junction nodes
    - tlLogic elements parsed into Vec of SignalPlan (returned separately)
    - Roundabout edges detected from <roundabout> element
    - Warning logged for unmapped attributes (e.g., spreadType, shape complexity)
    - Empty/missing file returns descriptive error
  </behavior>
  <action>
Add `quick-xml = "0.37"` to velos-net/Cargo.toml dependencies.

Create `tests/fixtures/simple.net.xml` -- a minimal valid SUMO network file with:
- 3 junctions (j0, j1, j2) with x,y positions
- 4 edges (e0, e1, e2, e3) each with 1-2 lanes (id, speed, length, shape)
- 2 internal edges (:j1_0, :j1_1) that must be filtered
- 4 connections mapping edges to successor edges at junctions
- 1 tlLogic with 2 phases (green/red cycle)
- 1 roundabout element referencing edge IDs

Create `crates/velos-net/src/sumo_import.rs`:
- `pub struct SumoNetImporter;`
- `pub fn import_sumo_net(path: &Path) -> Result<(RoadGraph, Vec<SumoSignalPlan>, Vec<String>), NetError>`
  - Returns: (graph, signal_plans, warnings_list)
  - Uses `quick-xml` streaming reader (not DOM) for memory efficiency with large files
  - Parse elements in order: `<edge>` with `<lane>` children, `<junction>`, `<connection>`, `<tlLogic>` with `<phase>`, `<roundabout>`
  - Skip internal edges: `id.starts_with(':')`
  - Map SUMO edge attributes to RoadEdge: numLanes -> lanes, speed -> speed_limit, length -> length, from/to -> graph connectivity
  - Map junction x,y to RoadNode
  - Map connections to graph edge adjacency
  - Map tlLogic to `SumoSignalPlan { junction_id: String, phases: Vec<SignalPhase> }`
  - For every unmapped attribute encountered, push a warning string to the warnings list. Per user decision: NEVER silently drop attributes.
  - Map SUMO edge types (highway.primary, etc.) to RoadClass using best-effort heuristic

Create `SumoSignalPlan` struct that wraps the velos-signal SignalPlan with the SUMO junction ID mapping.

Add `pub mod sumo_import;` to lib.rs.

Write tests in `crates/velos-net/tests/sumo_net_import.rs` using the fixture file.
  </action>
  <verify>
    <automated>cargo test -p velos-net --test sumo_net_import</automated>
  </verify>
  <done>SUMO .net.xml import produces valid RoadGraph. Internal edges filtered. Signal plans parsed. Unmapped attributes produce warnings. All tests pass.</done>
</task>

- `pub struct SumoVehicleParams` with fields: accel (f64), decel (f64), max_speed (f64), sigma (Option<f64>), min_gap (f64), length (f64), car_follow_model (CarFollowModelType)

  - Uses quick-xml streaming reader

  - Map SUMO carFollowModel attribute: "Krauss" (default) -> Krauss, "IDM" -> IDM, anything else -> Other(name) with warning
  - Push warning for every unmapped attribute
