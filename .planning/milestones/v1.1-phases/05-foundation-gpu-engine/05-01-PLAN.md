<task type="auto" tdd="true">
  <name>Task 1: Fixed-point arithmetic types (Q16.16, Q12.20, Q8.8)</name>
  <files>crates/velos-core/src/fixed_point.rs, crates/velos-core/src/lib.rs, crates/velos-core/tests/fixed_point_tests.rs</files>
  <behavior>
    - FixPos (Q16.16): f64 -> FixPos -> f64 round-trip within 1/65536 tolerance
    - FixSpd (Q12.20): f64 -> FixSpd -> f64 round-trip within 1/1048576 tolerance
    - FixLat (Q8.8): f64 -> FixLat -> f64 round-trip within 1/256 tolerance
    - fix_mul_q16(FixPos(1.0), FixPos(2.0)) == FixPos(2.0)
    - fix_mul_q16 handles max position (65535m) without overflow
    - fix_mul_q16 handles negative values correctly
    - FixPos + FixPos addition works correctly
    - Speed * dt position update: FixSpd(13.89) * FixPos(0.1) produces correct displacement
  </behavior>
  <action>
Create `crates/velos-core/src/fixed_point.rs` with three newtype wrappers around i32:
- `FixPos` (Q16.16): 16 integer bits, 16 fractional bits. Range: [-32768, 32767] metres. Resolution: ~0.015mm.
- `FixSpd` (Q12.20): 12 integer bits, 20 fractional bits. Range: [-2048, 2047] m/s. Resolution: ~0.001 mm/s.
- `FixLat` (Q8.8): 8 integer bits, 8 fractional bits. Range: [-128, 127] metres. Resolution: ~3.9mm.

Each type needs:
- `from_f64(v: f64) -> Self` and `to_f64(self) -> f64` conversions
- `from_f32(v: f32) -> Self` and `to_f32(self) -> f32` conversions (for GPU interop)
- Implement `Add`, `Sub`, `Neg` for same-type arithmetic
- Implement `PartialOrd`, `Ord` (integer comparison is correct for same-format fixed-point)
- Derive `Copy, Clone, Debug, PartialEq, Eq, bytemuck::Pod, bytemuck::Zeroable` with `#[repr(transparent)]`

Create `fix_mul_q16(a: i32, b: i32) -> i32` free function using the 16-bit half splitting technique from RESEARCH.md to avoid i64 overflow. Handle sign separately. This is the critical function -- test thoroughly.

Create `fix_mul_mixed(pos: FixPos, spd: FixSpd, dt_pos: FixPos) -> FixPos` for the common `position += speed * dt` update pattern. This requires cross-format multiplication (Q12.20 * Q16.16 -> Q16.16).

Add `pub mod fixed_point;` to lib.rs. Re-export key types.

Write tests in `crates/velos-core/tests/fixed_point_tests.rs` covering all behavior items above.
  </action>
  <verify>
    <automated>cargo test -p velos-core --test fixed_point_tests</automated>
  </verify>
  <done>All three fixed-point types exist with conversions and arithmetic. Multiplication handles edge cases without overflow. All tests pass.</done>
</task>

  <files>crates/velos-vehicle/src/krauss.rs, crates/velos-vehicle/src/lib.rs, crates/velos-vehicle/tests/krauss_tests.rs, crates/velos-core/src/components.rs</files>
  <behavior>

    - krauss_safe_speed with zero gap returns 0.0 (stopped)

    - krauss_dawdle reduces speed by [0, sigma*accel] range (never increases speed)
    - krauss_dawdle with sigma=0.0 returns speed unchanged
    - krauss_update produces (velocity, displacement) tuple with v >= 0
    - KraussParams::sumo_default() returns sigma=0.5, accel=2.6, decel=4.5, tau=1.0, max_speed=13.89, min_gap=2.5
    - CarFollowingModel::IDM == 0u8, CarFollowingModel::Krauss == 1u8
  </behavior>
  <action>
Create `crates/velos-vehicle/src/krauss.rs` following the exact code from RESEARCH.md code examples section:
- `KraussParams` struct with SUMO-faithful defaults (sigma=0.5, accel=2.6, decel=4.5, tau=1.0, max_speed=13.89 m/s, min_gap=2.5m)
- `krauss_safe_speed(params, gap, leader_speed, own_speed) -> f64` using the SUMO vsafe formula
- `krauss_dawdle(speed, params, rng) -> f64` applying random deceleration proportional to sigma
- `krauss_update(params, own_speed, gap, leader_speed, dt, rng) -> (f64, f64)` full velocity update

Follow the same pattern as idm.rs: pure functions with params struct, f64 precision, no ECS dependency.

Per user decision: no hardcoded default model per vehicle type. CarFollowingModel is a standalone ECS component tag, not embedded in VehicleType.

Write tests in `crates/velos-vehicle/tests/krauss_tests.rs` covering all behavior items. Use `rand::rngs::StdRng` with fixed seed for dawdle tests.
