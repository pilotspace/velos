  <name>Task 1: Extend RoadGraph + 5-district OSM import + cleaning pipeline</name>
  <files>crates/velos-net/src/graph.rs, crates/velos-net/src/osm_import.rs, crates/velos-net/src/cleaning.rs, crates/velos-net/src/lib.rs, crates/velos-net/Cargo.toml, crates/velos-net/tests/cleaning_tests.rs, crates/velos-net/tests/hcmc_rules_tests.rs, data/hcmc/overrides.toml</files>
  <behavior>
    - Cleaning: graph with 3 disconnected components keeps only the largest
    - Cleaning: edges shorter than 5m are merged into adjacent edges
    - Cleaning: lane count inferred from road class when missing (Primary=3, Secondary=2, Tertiary=2, Residential=1, Service=1)
    - HCMC rules: alleys <4m wide tagged as motorbike-only
    - HCMC rules: time-dependent one-way edges have TimeWindow with direction per time slot
    - Override: TOML file with edge/junction corrections applied after cleaning
    - Serialization: cleaned graph round-trips through binary format
  </behavior>
  <action>
**Extend RoadEdge** in graph.rs:
- Add `motorbike_only: bool` field
- Add `time_windows: Option<Vec<TimeWindow>>` field for time-dependent one-way edges
- Create `TimeWindow` struct: `{ start_hour: u8, end_hour: u8, direction: OneWayDirection }` where `OneWayDirection` is `Forward | Reverse | Both`
- Add `Motorway` and `Trunk` variants to RoadClass if not already present (check existing)
- Add `serialize_binary(&self, path: &Path) -> Result<()>` and `deserialize_binary(path: &Path) -> Result<RoadGraph>` using `postcard` (safer choice -- check if oxicode is available on crates.io; if not, use postcard). Add `serde::{Serialize, Deserialize}` derives to RoadNode, RoadEdge, RoadGraph.

**Extend OSM import** in osm_import.rs:
- Support 5-district bounding box (Districts 1, 3, 5, 10, Binh Thanh). The bounding box should encompass all 5 districts.
- Add `import_osm_5district(path: &Path) -> Result<RoadGraph, NetError>` function that uses a wider bounding box
- Parse additional OSM tags: `motorcycle=designated`, `motorcycle=yes`, `junction=roundabout`
- Detect alleys: `highway=service` or `highway=residential` with `width < 4.0` tag -> `motorbike_only = true`

**Create cleaning.rs** with `clean_network(graph: &mut RoadGraph) -> CleaningReport`:
1. `remove_small_components(graph, min_edges: 10)` -- keep largest strongly connected component, remove others
2. `merge_short_edges(graph, min_length: 5.0)` -- merge edges shorter than 5m into adjacent edge (combine geometry, sum length)
3. `infer_lane_counts(graph)` -- set lane count from road class when lanes==0
4. `apply_overrides(graph, overrides: &OverrideFile)` -- apply corrections from TOML file
5. `tag_motorbike_only_lanes(graph)` -- OSM tags + road width heuristic
6. `apply_time_dependent_oneways(graph)` -- mark known HCMC time-dependent one-way streets
7. `validate_connectivity(graph) -> bool` -- verify the final graph is strongly connected

`CleaningReport` struct tracks counts of each operation.

**Create override file** at `data/hcmc/overrides.toml`:
```toml
# HCMC OSM Override File
# Corrections to OSM data for edges/junctions where OSM is wrong.
# Each override documents the reasoning.

# Example format (start empty, users add overrides as needed):
# [[edge_override]]
# edge_id = "way/123456"
# lanes = 3
# reason = "Field survey shows 3 lanes, OSM has 2"
```

Add dependencies to velos-net/Cargo.toml: `serde = { workspace = true, features = ["derive"] }`, `toml = "0.8"`, `postcard = { version = "1", features = ["alloc"] }`, `rayon = "1"` (for parallel cleaning operations). Add `petgraph` serde feature if available, otherwise implement manual serialization.

Add `pub mod cleaning;` to lib.rs.
  </action>

  </verify>
  <done>Cleaning pipeline removes disconnected components, merges short edges, infers lanes, applies overrides. Motorbike-only detection works. Time-dependent one-way edges have TimeWindow. Binary serialization round-trips. All tests pass.</done>
</task>
