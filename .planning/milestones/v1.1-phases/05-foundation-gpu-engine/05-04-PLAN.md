Create `crates/velos-gpu/shaders/wave_front.wgsl` with the wave-front dispatch shader:

Buffer layout:
```wgsl
struct Params {
    agent_count: u32,
    dt: f32,
    step_counter: u32,
    _pad: u32,
}

struct AgentState {
    edge_id: u32,
    lane_idx: u32,
    position: i32,      // Q16.16
    lateral: i32,       // Q8.8 in i32
    speed: i32,         // Q12.20
    acceleration: i32,
    cf_model: u32,      // 0=IDM, 1=Krauss
    rng_state: u32,
}

@group(0) @binding(0) var<uniform> params: Params;
@group(0) @binding(1) var<storage, read_write> agents: array<AgentState>;
@group(0) @binding(2) var<storage, read> lane_offsets: array<u32>;
@group(0) @binding(3) var<storage, read> lane_counts: array<u32>;
@group(0) @binding(4) var<storage, read> lane_agents: array<u32>;
```

Wave-front kernel (`@compute @workgroup_size(64)`):
- Each workgroup processes one lane (workgroup_id.x = lane_idx)
- Only thread 0 (local_invocation_id.x == 0) is active
- Process agents sequentially front-to-back (leader first, already sorted by position descending)
- For each agent: read leader's ALREADY-UPDATED position/speed from current step
- Compute gap and delta_v from leader
- Branch on `cf_model`:
  - `CF_IDM (0)`: IDM acceleration using safe_pow4, v_eff=max(v,0.1), gap_eff=max(gap,0.01)
  - `CF_KRAUSS (1)`: Krauss safe speed + dawdle using PCG hash RNG
- Update speed and position in-place (wave-front: subsequent agents see updated values)
- Clamp speed >= 0

Include PCG hash RNG functions from RESEARCH.md code examples:
```wgsl
fn pcg_hash(input: u32) -> u32;
fn rand_float(agent_id: u32, step: u32) -> f32;
```

Use fixed-point for position and speed updates. Use f32 intermediates for IDM/Krauss calculations (compute accel in f32, then convert result to fixed-point). This avoids the full fixed-point perf penalty in the inner loop while keeping position/speed storage deterministic.

Use workgroup_size(64) not 256 per RESEARCH.md Pitfall 2 guidance (only thread 0 works, smaller workgroup wastes fewer threads).

IDM implementation in WGSL should match the CPU idm.rs logic:
- `safe_pow4(x) = x*x*x*x` (never `pow(x, 4.0)`)
- `v_eff = max(v, 0.1)` kickstart
- `gap_eff = max(gap, 0.01)` floor
- Accel clamped to `[-9.0, a_max]`

Krauss implementation in WGSL should match the CPU krauss.rs logic:
- `v_safe = leader_speed + (gap - leader_speed * tau) / ((leader_speed + own_speed) / (2*b) + tau)`
- `v_desired = min(own_speed + accel * dt, max_speed)`
- `v_next = min(v_desired, v_safe)`
- Dawdle: `v_dawdled = v_next - sigma * min(v_next, accel) * rand_float(agent_id, step)`

Run `naga --validate` on both shaders after creation.
