  <name>Task 1: METIS graph partitioning + boundary agent protocol</name>
  <files>crates/velos-gpu/src/partition.rs, crates/velos-gpu/src/multi_gpu.rs, crates/velos-gpu/src/lib.rs, crates/velos-gpu/Cargo.toml, crates/velos-gpu/tests/boundary_protocol_tests.rs</files>
  <behavior>
    - partition_network with k=2 on a 100-edge graph -> 2 partitions, each with ~50 edges (+/- 10% balance)
    - partition_network with k=4 on a 100-edge graph -> 4 partitions with minimized edge cuts
    - Same network + same k -> same partition assignment (deterministic)
    - BoundaryAgent created when agent crosses partition boundary edge
    - Outbox agent from partition A appears in inbox of partition B next step
    - Agent state (position, speed, cf_model, rng_state) preserved across transfer
    - MultiGpuScheduler.step() processes: drain inbox -> dispatch -> collect outbox -> route
    - Single-GPU logical partitions: each partition has own buffers but shares Device/Queue
  </behavior>
  <action>
Add `metis = "0.2"` to velos-gpu/Cargo.toml. Note: metis crate requires libmetis. If the crate fails to build (vendored build may fail on macOS), fall back to a simple balanced graph bisection using petgraph's connected components + BFS-based balanced cut. Document the fallback.

Create `crates/velos-gpu/src/partition.rs`:
- `pub struct PartitionAssignment` with fields: `edge_to_partition: HashMap<u32, u32>`, `boundary_edges: Vec<(u32, u32, u32)>` (edge_id, src_partition, dst_partition), `partition_count: u32`
- `pub fn partition_network(graph: &RoadGraph, k: u32) -> PartitionAssignment`
  - Convert petgraph to METIS adjacency format (CSR)
  - Call `metis::Graph::new(1, k, &xadj, &adjncy).part_kway(&mut part)` (or fallback)
  - Identify boundary edges: edges where source and target nodes are in different partitions
  - Build partition assignment mapping
- `pub fn partition_edges(assignment: &PartitionAssignment, partition_id: u32) -> Vec<u32>` returns edge IDs for a partition

Create `crates/velos-gpu/src/multi_gpu.rs`:
- `pub struct BoundaryAgent` with fields matching GpuAgentState plus `dest_partition: u32`, `dest_edge_id: u32`
- `pub struct GpuPartition`:
  - `partition_id: u32`
  - `edge_ids: Vec<u32>` -- edges in this partition
  - `agent_states: Vec<GpuAgentState>` -- agents on edges in this partition
  - `lane_data: (Vec<u32>, Vec<u32>, Vec<u32>)` -- lane_offsets, lane_counts, lane_agents
  - `outbox: Vec<BoundaryAgent>` -- agents that crossed boundary edges this step
  - `inbox: Vec<BoundaryAgent>` -- agents arriving from other partitions
  - Methods: `spawn_inbox_agents()`, `collect_outbox_agents(boundary_map)`, `dispatch(dispatcher, encoder)`
- `pub struct MultiGpuScheduler`:
  - `partitions: Vec<GpuPartition>`
  - `boundary_map: HashMap<u32, (u32, u32)>` -- edge_id -> (src_partition, dst_partition)
  - `pub fn new(assignment: PartitionAssignment, device: &wgpu::Device) -> Self`
  - `pub fn step(&mut self, dispatcher: &ComputeDispatcher, encoder: &mut CommandEncoder, dt: f32)`:
    1. Drain inboxes: spawn boundary agents on destination partitions
    2. Sort agents by lane within each partition (rayon parallel)
    3. Upload lane data + agent states per partition
    4. Dispatch wave_front shader for each partition (sequential dispatches with barriers, or concurrent if wgpu supports)
    5. Readback agent states per partition
    6. Collect outboxes: identify agents on boundary edges whose next edge is in a different partition
    7. Route outbox agents to correct partition inboxes
  - `pub fn agent_count(&self) -> usize` total across all partitions

Per user decision: "Multi-GPU validated via simulated partitions on single GPU (2-4 logical partitions with own buffers, real inbox/outbox boundary agent protocol)". Each GpuPartition shares the same wgpu::Device/Queue but has its own GPU buffer allocations.

Add `pub mod partition;` and `pub mod multi_gpu;` to lib.rs.

Write tests in `crates/velos-gpu/tests/boundary_protocol_tests.rs`:
- Test METIS partition balance on a synthetic 100-edge grid graph
- Test boundary agent outbox/inbox transfer for an agent crossing partition boundary
- Test that agent state is preserved across transfer (all fields match)
- Test MultiGpuScheduler.step() with 2 partitions on a small graph (20 agents)
- These tests can be CPU-only (test the protocol logic, not GPU dispatch)
