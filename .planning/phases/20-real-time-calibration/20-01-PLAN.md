<objective>
Refactor calibration from fixed 300-second timer to event-driven window-change detection with stability safeguards.

Purpose: CAL-02 requires continuous calibration from streaming detection data. The existing batch timer fires every 300s regardless of data arrival. The new trigger reacts to aggregation window completion, making demand adjustments responsive to real data flow.

Output: Refactored step_calibration() with window-change trigger, extended CameraCalibrationState with staleness tracking, stability safeguards (min observation threshold, decay toward baseline, per-step change cap), and calibration pause flag.
</objective>

    - `pub last_window_start_ms: i64` (default -1)

    Add constant `MIN_OBSERVED_THRESHOLD: u32 = 10` (minimum observed count per camera to participate in calibration).

    Modify compute_camera_ratio to skip calibration when `observed < MIN_OBSERVED_THRESHOLD` (return previous_ratio, same as simulated-below-threshold behavior).

    - Applies `state.previous_ratio += (1.0 - state.previous_ratio) * decay.min(1.0)` -- always moves toward 1.0 regardless of direction

    - If no old_factor exists for an OD pair, leave the new factor uncapped (first calibration)
    - Per user decision: cap constant is MAX_FACTOR_CHANGE_PER_STEP: f32 = 0.2

    Update Default impl for CameraCalibrationState to include new fields.

    In sim.rs, add three new fields to SimWorld:
    - `pub(crate) calibration_paused: bool` (default false) -- user toggle to freeze calibration
    - `pub(crate) last_processed_windows: HashMap<u32, i64>` (default empty) -- per-camera last processed window start_ms

    In sim_calibration.rs, refactor step_calibration():
    1. Remove `CALIBRATION_INTERVAL_SECS` constant entirely
    2. Add `CALIBRATION_COOLDOWN_SECS: f64 = 30.0` constant
    3. Early return if `self.calibration_paused`
    4. Lock registry, get camera list, early return if empty
    5. Lock aggregator, check for ANY camera having a new window (latest_window.start_ms > last_processed_windows[cam_id])
    6. If no new windows found, increment `consecutive_stale_windows` for each camera whose window is unchanged, call `decay_toward_baseline` for each, drop locks, return
    7. Apply cooldown: if `self.sim_time - self.last_calibration_time < CALIBRATION_COOLDOWN_SECS`, return (new windows exist but cooldown not elapsed)
    8. Capture current overlay factors BEFORE computing new ones (for change cap)
    9. Proceed with existing calibration logic (simulated counts ECS query, compute_calibration_factors)
    10. Call `apply_change_cap(&old_factors, &mut new_overlay)` before swapping
    11. Update `self.last_processed_windows` for all cameras that have windows
    12. Reset `consecutive_stale_windows = 0` for cameras that had new windows
    13. Log at info level on recalibration: "Calibration triggered: {N} cameras, {M} OD factors updated"

    Lock ordering: always registry first, then aggregator. Drop registry before ECS query (existing pattern). Re-acquire for compute_calibration_factors.

    IMPORTANT: The staleness check (step 6) must track which cameras have NEW windows vs UNCHANGED windows. Cameras with new windows get consecutive_stale_windows reset to 0. Cameras with unchanged windows get incremented. This happens regardless of whether cooldown blocks the full recalibration.
