1. Modify `crates/velos-gpu/src/sim_render.rs` to add camera overlay rendering:
   - Add `build_camera_instances(cameras: &[&Camera], projection: &EquirectangularProjection, show_cameras: bool) -> Vec<Instance>`:
     - If !show_cameras, return empty vec
     - For each camera: convert lat/lon to local meters via projection
     - Camera icon: small diamond or circle quad (4 vertices) at camera position, distinct color (e.g., yellow)
     - FOV cone: triangle fan from camera position to arc at range_m distance
       - Use simple triangle approximation: camera_pos, left_edge_point, right_edge_point
       - left_edge = cam_pos + range_m * (heading - fov/2), right_edge = cam_pos + range_m * (heading + fov/2)
       - Semi-transparent fill (alpha ~0.15) for coverage area, brighter outline (alpha ~0.6)
     - Use existing Instance struct / quad strip pattern from guide line rendering (Phase 16 established quad strip for overlays)
   - Wire into the render pipeline: after road rendering, before agent rendering
   - Read camera list from camera_registry (Arc<Mutex<CameraRegistry>>) with brief lock
   - Toggle controlled by `show_cameras: bool` field on SimWorld (connected to egui checkbox added in Plan 03)

2. Create `crates/velos-api/tests/grpc_integration.rs`:
   - Starts a tonic server in-process on a random port using `tokio::net::TcpListener`
   - Tests:
     - `test_register_camera`: RegisterCamera RPC returns camera_id > 0 and covered_edge_ids list
     - `test_list_cameras`: After registering 2 cameras, ListCameras returns both
     - `test_stream_detections`: Client streams 3 DetectionBatch messages, receives 3 DetectionAck with OK status
     - `test_unknown_camera_detection`: Stream detection for unregistered camera_id, receive UNKNOWN_CAMERA ack
   - Use a minimal in-memory RTree with a few test edges for camera registration
   - Run with: `cargo test -p velos-api --test grpc_integration`

3. Create `tools/python/detection_client.py`:
   - Generate Python stubs from proto first (document the command in a comment at top):
     `python -m grpc_tools.protoc --proto_path=proto --python_out=tools/python --pyi_out=tools/python --grpc_python_out=tools/python proto/velos/v2/detection.proto`
   - VelosDetectionClient class:
     - `__init__(self, addr: str = "localhost:50051")`
     - `register_camera(self, lat, lon, heading_deg, fov_deg, range_m, name) -> (camera_id, covered_edges)`
     - `list_cameras(self) -> list[CameraInfo]`
     - `stream_detections(self, batches: Iterable[DetectionBatch]) -> Iterator[DetectionAck]`
     - Helper: `make_detection_event(camera_id, vehicle_class, count, speed_kmh=None) -> DetectionEvent`
   - Import generated stubs from `velos.v2.detection_pb2` and `velos.v2.detection_pb2_grpc`

4. Create `tools/python/test_detection_client.py`:
   - Requires running VELOS gRPC server (not automated in CI -- manual integration test)

     - List cameras and verify the registered camera appears
     - Stream 5 detection batches with mixed vehicle classes
     - Print received acks and aggregation summary
   - Run with: `python tools/python/test_detection_client.py`
   - Include argparse for --addr flag (default localhost:50051)

Note: Python stub generation requires `grpcio-tools` installed via `uv pip install grpcio grpcio-tools`. Add a requirements.txt at tools/python/requirements.txt with these dependencies.
