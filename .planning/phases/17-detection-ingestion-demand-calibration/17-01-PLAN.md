  <name>Task 2: Implement error types and async-sync bridge</name>

  <action>
1. Create `crates/velos-api/src/error.rs`:
   - `ApiError` enum with thiserror derive:
     - `UnknownCamera(u32)` -- camera_id not registered
     - `InvalidData(String)` -- malformed detection event
     - `ChannelFull` -- bridge channel at capacity
     - `ChannelClosed` -- simulation side disconnected
   - Implement `From<ApiError>` for `tonic::Status` mapping (UnknownCamera -> NOT_FOUND, InvalidData -> INVALID_ARGUMENT, Channel* -> UNAVAILABLE)

2. Create `crates/velos-api/src/bridge.rs`:
   - `ApiCommand` enum:
     - `RegisterCamera { request: RegisterCameraRequest, reply: tokio::sync::oneshot::Sender<RegisterCameraResponse> }` -- oneshot for request-response
     - `DetectionBatch { batch: DetectionBatch }` -- fire-and-forget to aggregator
   - `ApiBridge` struct:
     - `cmd_tx: tokio::sync::mpsc::Sender<ApiCommand>` (async sender for gRPC handlers)
     - `cmd_rx: tokio::sync::mpsc::Receiver<ApiCommand>` (sync receiver for SimWorld frame loop via try_recv)
   - `ApiBridge::new(capacity: usize) -> (ApiBridge, mpsc::Sender<ApiCommand>)` -- returns the bridge (with rx) and a cloneable tx for the gRPC service
   - Use the generated proto types from `crate::proto::velos::v2::*`
   - Channel capacity: 256 (bounded, per research anti-pattern guidance)

3. Add unit tests in each file:
   - error.rs: test ApiError -> tonic::Status conversion for each variant
   - bridge.rs: test channel creation, send/try_recv round-trip, full channel behavior
