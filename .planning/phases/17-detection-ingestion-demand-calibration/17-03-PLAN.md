    - Test: swap replaces overlay atomically, current() reads latest
    - Test: clone_handle shares same underlying ArcSwap
    - Test: compute_calibration_factors with observed=100, simulated=80 -> ratio 1.25
    - Test: ratio clamped to 2.0 when observed >> simulated
    - Test: ratio clamped to 0.5 when observed << simulated
    - Test: simulated count of 0 defaults ratio to 1.0 (no adjustment)
    - Test: simulated count below threshold (<=5) defaults ratio to 1.0
    - Test: EMA smoothing: previous ratio 1.0, raw ratio 1.5, alpha=0.3 -> new ratio = 0.3*1.5 + 0.7*1.0 = 1.15
    - Test: EMA applied before clamping
  </behavior>
  <action>
1. Implement `crates/velos-api/src/calibration.rs`:
   - `CalibrationOverlay` struct:
     - `factors: HashMap<(Zone, Zone), f32>` -- per OD-pair multiplicative scaling factor
     - `timestamp_sim_seconds: f64` -- when this overlay was computed
   - `CalibrationStore` struct mirroring PredictionStore:
     - `inner: Arc<ArcSwap<CalibrationOverlay>>`
     - `new() -> Self` -- starts with empty factors, timestamp 0.0
     - `current(&self) -> Guard<Arc<CalibrationOverlay>>` -- lock-free read
     - `swap(&self, new: CalibrationOverlay)` -- atomic replace
     - `clone_handle(&self) -> Self` -- cheap Arc clone
   - `CameraCalibrationState` struct (per-camera state for ratio tracking):
     - `previous_ratio: f32` -- for EMA smoothing, starts at 1.0
     - `last_observed: u32`
     - `last_simulated: u32`
   - `compute_calibration_factors(cameras: &CameraRegistry, aggregator: &DetectionAggregator, simulated_counts: &HashMap<u32, HashMap<VehicleClass, u32>>, camera_states: &mut HashMap<u32, CameraCalibrationState>, od_matrix: &OdMatrix, graph: &RoadGraph) -> CalibrationOverlay`:
     - For each camera: total_observed = sum of all class counts in latest window, total_simulated = sum from simulated_counts
     - If simulated <= 5: ratio = 1.0 (skip)
     - Else: raw_ratio = observed / simulated
     - EMA: smoothed = 0.3 * raw_ratio + 0.7 * previous_ratio (update camera_state)
     - Clamp to [0.5, 2.0]
     - Map camera ratio to OD pairs: for each OD pair, if any edge in the pair's route passes through camera's covered_edges, apply that camera's ratio
     - Note: For this initial implementation, use a simplified heuristic -- apply camera ratio to all OD pairs whose origin or destination zone contains a covered edge. Full route-based mapping is a future optimization.
     - Return CalibrationOverlay with factors and current sim time
   - Unit tests per behavior list
  </action>
