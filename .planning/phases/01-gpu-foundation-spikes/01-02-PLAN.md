Validate the ECS-to-GPU round-trip pipeline: project 1K hecs entities into SoA GPU buffers, dispatch a pass-through compute shader, read back results, and verify bitwise equality. This is Track C from the spike ordering -- the GO/NO-GO gate that proves the fundamental pipeline works before investing in wave-front dispatch and PCG hash.

Purpose: If this round-trip fails (data corruption, alignment mismatch, buffer stride errors), the entire GPU compute approach needs redesign. Succeeding here validates the core data path that every future simulation step will use.

Output: Working ECS-to-GPU projection, GPU dispatch, GPU-to-CPU readback, verified with 1K agents. GO/NO-GO gate: if this plan's tests pass, proceed to Plans 03 and 04. If they fail, stop and redesign buffer layout.
