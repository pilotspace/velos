  <behavior>
    - 20 lanes x 50 agents: each agent reads its leader's position and computes gap (leader.offset - self.offset)
    - Wave-front order: agent at position i reads agent at position i-1 (the leader ahead in the lane)
    - After dispatch: all gaps >= 0, no agent has overtaken its leader, positions are monotonically decreasing in agent order within each lane
    - CPU reference: same gap computation in Rust produces identical results to GPU shader
    - Lane-change dual-leader: agent with LaneChangeState reads leaders from BOTH source_lane and target_lane, uses conservative (smaller) gap
    - Empty target lane during lane-change: gap defaults to "infinite" (large value), only source lane leader constrains
    - Lane-change at progress=1.0: agent fully in target lane, only reads target lane leader
    - Agent count not a multiple of 256: shader correctly handles bounds checking (last workgroup has fewer than 256 active invocations)
  </behavior>
  <action>
    1. Create crates/velos-gpu/shaders/wave_front.wgsl:
       - Include emulated i64 functions (copy from fixed_point.wgsl -- WGSL has no includes)
       - Struct WaveParams { agent_count: u32, lane_count: u32, agents_per_lane: u32, padding: u32 }
       - Storage buffers: positions (read_write array of GpuPosition), kinematics (read_write array of GpuKinematics), lane_change (read array of GpuLaneChangeState { source_lane: u32, target_lane: u32, progress: f32, padding: u32 }), leader_indices (read array of i32 -- index of the leader agent for each agent, -1 if no leader), gaps (write array of i32 -- computed gap in Q16.16 for verification)
       - Define GpuLaneChangeState struct: { source_lane: u32, target_lane: u32, progress: f32, _pad: u32 } -- 16 bytes
       - Pre-computation (CPU-side): sort agents by lane, then by descending position within each lane. Compute leader_indices array: for agent at sorted position i in lane L, leader is agent at sorted position i-1 in the same lane (or -1 for the first agent in the lane).
       - Compute kernel @compute @workgroup_size(256) fn wave_front_update:
         - Guard: if (gid.x >= params.agent_count) { return; }
         - Read this agent's position and lane_change state
         - If lane_change.progress > 0.0 and < 1.0 (transitioning): find leader in source_lane AND target_lane, compute both gaps, use min (conservative)
         - If lane_change.progress >= 1.0 or == 0.0: use single leader from leader_indices
         - Compute gap = leader.offset - self.offset (Q16.16 subtraction)
         - Write gap to gaps output buffer
         - Apply a minimal gap-based deceleration as proof of concept: if gap < safe_gap, reduce speed. This proves the wave-front ordering matters (if order were wrong, stale leader data would produce different results than CPU reference).

    2. Create CPU reference implementation in crates/velos-gpu/tests/gpu_wave_front.rs:
       - #[cfg(feature = "gpu-tests")] gated

Agents in each lane are placed at evenly spaced positions (e.g., lane L has agents at offsets 1000, 900, 800... in Q16.16). Some agents have active lane-change states.
       - CPU reference function: cpu_wave_front(positions, kinematics, lane_changes, leader_indices) -> Vec<i32> (gaps) that mirrors the shader logic exactly in Rust.

    3. Test: test_wave_front_matches_cpu_reference:

       - Run CPU reference to get expected gaps
       - Upload data to GPU, dispatch wave_front shader, readback gaps
       - Assert GPU gaps match CPU gaps bitwise for all 1000 agents

    4. Test: test_wave_front_invariants:
       - After dispatch: for each lane, verify positions are monotonically ordered
       - For each agent with a leader: verify gap >= 0
       - No agent has overtaken its leader (self.offset < leader.offset in forward direction)

    5. Test: test_dual_leader_lane_change:

    7. Test: test_wave_front_non_multiple_256:

  <action>
    1. Create crates/velos-gpu/shaders/pcg_hash.wgsl:
       - fn pcg_hash(input: u32) -> u32: Jarzynski & Olano constants. state = input * 747796405u + 2891336453u; word = ((state >> ((state >> 28u) + 4u)) ^ state) * 277803737u; return (word >> 22u) ^ word;
       - fn agent_random(agent_id: u32, step: u32) -> f32: seed = agent_id ^ (step * 1664525u + 1013904223u); hash = pcg_hash(seed); return f32(hash) / 4294967295.0;
       - Validation kernel @compute @workgroup_size(256) fn validate_pcg:
         - Params: { count: u32, step: u32 }
         - Input: array of agent_ids (u32)
         - Output: array of hash results (u32) and array of float results (f32)
         - For each invocation: output_hash[gid.x] = pcg_hash(input_seed); output_float[gid.x] = agent_random(agent_ids[gid.x], step);

    2. Create CPU reference in crates/velos-gpu/tests/gpu_pcg_hash.rs:
       - fn cpu_pcg_hash(input: u32) -> u32: identical algorithm in Rust using wrapping_mul and wrapping_add
